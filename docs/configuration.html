Configuration

Broker Configuration

Environment Variables

Variable Default Description
BROKER_ID 1 Unique broker identifier (1-999)
LISTEN_ADDR 0.0.0.0 Address to listen on
PORT 9092 Broker protocol port
DATA_DIR ./data Directory for partition data
LOG_LEVEL info Logging level (debug, info, warn, error)
METRICS_PORT 9101 Prometheus metrics port
REPLICATION_FACTOR 3 Default replication factor for topics
MIN_INSYNC_REPLICAS 2 Min replicas that must ACK writes
RETENTION_MS 604800000 Default message retention (7 days)
SEGMENT_SIZE 1073741824 Max segment file size (1GB)
COMPRESSION_TYPE none Default compression (none, gzip, snappy, lz4)

Configuration File (YAML)

broker:
  id: 1
  listen_addr: 0.0.0.0
  port: 9092
  rack: us-east-1a

storage:
  data_dir: /data/streamforge
  segment_size: 1073741824  # 1GB
  index_size: 10485760       # 10MB
  compression_type: snappy
  sync_interval_ms: 5000

replication:
  min_insync_replicas: 2
  default_replication_factor: 3
  replication_timeout_ms: 30000

retention:
  retention_ms: 604800000    # 7 days
  retention_bytes: 0          # 0 = unlimited

network:
  max_connections: 5000
  request_timeout_ms: 30000
  num_network_threads: 8
  socket_send_buffer: 102400
  socket_receive_buffer: 102400

metrics:
  enabled: true
  port: 9101
  interval_ms: 60000

logging:
  level: info
  format: json
  output: stdout

Setting Configuration

# Via environment variables
export BROKER_ID=1
export PORT=9092
export DATA_DIR=/data/streamforge
./broker

# Via configuration file
./broker --config broker-config.yml

# Via command line flags
./broker --broker-id 1 --port 9092 --data-dir /data

Producer Configuration (Go SDK)

ProducerConfig Structure

type ProducerConfig struct {
  Brokers          []string      // Broker addresses
  Topic            string        // Default topic
  Compression      CompressionType
  BatchSize        int           // Messages per batch
  BatchTimeoutMs   int           // Max wait before sending
  MaxInFlight      int           // Max unacked batches
  Retries          int           // Retry attempts
  RetryBackoffMs   int           // Backoff between retries
  RequestTimeoutMs int           // Request timeout
  MaxMessageBytes  int           // Max message size
  RequiredAcks     int           // -1=all, 0=none, 1=leader
  ClientID         string        // Producer identifier
}

Default Configuration

config := producer.DefaultProducerConfig(
  []string{"localhost:9092", "localhost:9093", "localhost:9094"},
  "my-topic",
)

// Customize
config.BatchSize = 1000          // Batch 1000 messages
config.BatchTimeoutMs = 100      // Or 100ms, whichever first
config.Compression = Snappy      // Use Snappy compression
config.RequiredAcks = -1         // Wait for all replicas
config.Retries = 3               // Retry 3 times on failure

Configuration Values

Parameter Default Min Max Notes
batch_size 100 1 1000000 Higher = more throughput, more latency
batch_timeout_ms 100 0 60000 Max wait for batch to fill
max_in_flight 5 1 100 Concurrent unacked requests
required_acks 1 -1 1 -1=all, 0=none, 1=leader
max_message_bytes 1048576 1024 10485760 Max message size (1MB to 10MB)

Consumer Configuration (Go SDK)

ConsumerConfig Structure

type ConsumerConfig struct {
  Brokers           []string
  GroupID           string
  Topics            []string
  AutoCommit        bool
  AutoCommitMs      int
  SessionTimeoutMs  int
  HeartbeatMs       int
  MaxPollRecords    int
  OffsetResetPolicy string // earliest, latest, none
  EnablePartitionEOF bool
  FetchMinBytes     int
  FetchMaxWaitMs    int
}

Consumer Tuning

config := consumer.DefaultConsumerConfig(
  []string{"localhost:9092"},
  "my-group",
  []string{"my-topic"},
)

// Low-latency tuning
config.SessionTimeoutMs = 6000   // 6 seconds
config.HeartbeatMs = 2000        // 2 second heartbeat
config.FetchMaxWaitMs = 500      // Poll every 500ms
config.MaxPollRecords = 100      // Smaller batches

// High-throughput tuning
config.SessionTimeoutMs = 30000  // 30 seconds
config.HeartbeatMs = 10000       // 10 second heartbeat
config.FetchMaxWaitMs = 10000    // Poll less frequently
config.MaxPollRecords = 5000     // Larger batches
config.AutoCommit = true         // Auto-commit offsets

Storage Backend Configuration (C++)

struct LogEngineConfig {
  std::string data_dir;              // Data directory
  size_t segment_size;               // Max segment size
  size_t index_size;                 // Index file size
  bool compression_enabled;          // Enable compression
  CompressionType compression_type;  // Codec
  int sync_interval_ms;              // Flush interval
  bool use_mmap;                     // Memory-mapped I/O
  size_t cache_size;                 // Cache size
};

Configuration Example

LogEngineConfig config;
config.data_dir = "/data/streamforge";
config.segment_size = 1024 * 1024 * 1024;      // 1GB
config.compression_type = CompressionType::SNAPPY;
config.sync_interval_ms = 5000;                // Sync every 5s
config.use_mmap = true;                        // Enable mmap

LogEngine engine(config);

Client Configuration Best Practices

For Low Latency

// Producer: sacrifice throughput for latency
config.BatchSize = 10              // Small batches
config.BatchTimeoutMs = 10         // Short timeout
config.MaxInFlight = 1             // Sequential sends
config.Compression = None          // No compression overhead

// Consumer: poll frequently
config.MaxPollRecords = 100        // Process quickly
config.FetchMaxWaitMs = 100        // Check frequently
config.SessionTimeoutMs = 6000     // Detect failures quickly

For High Throughput

// Producer: sacrifice latency for throughput
config.BatchSize = 100000          // Large batches
config.BatchTimeoutMs = 1000       // Wait longer
config.MaxInFlight = 10            // Parallel sends
config.Compression = Snappy        // Compress for network

// Consumer: batch process
config.MaxPollRecords = 5000       // Process in batches
config.FetchMaxWaitMs = 5000       // Wait for data
config.AutoCommit = true           // Automatic offset management

For Resource Efficiency

// Broker
BROKER_ID=1
PORT=9092
DATA_DIR=/data
RETENTION_MS=86400000              # 1 day retention
SEGMENT_SIZE=536870912             # 512MB segments
COMPRESSION_TYPE=lz4               # Fast compression

// Consumer
config.SessionTimeoutMs = 30000    # Longer timeout
config.HeartbeatMs = 10000         # Less frequent
config.MaxPollRecords = 500        # Smaller batches

Monitoring Configuration

Prometheus Scrape Configuration

scrape_configs:
  - job_name: 'streamforge'
    scrape_interval: 15s
    scrape_timeout: 10s
    static_configs:
      - targets: ['localhost:9101']
    relabel_configs:
      - source_labels: [__address__]
        target_label: instance

Alert Rules

alert:HighLatency
  expr: broker_operation_latency_ms{quantile="0.99"} > 50
  for: 5m
  annotations:
    summary: "High latency on {{ $labels.instance }}"

alert:LowThroughput
  expr: rate(broker_messages_produced[5m]) < 1000
  for: 10m
  annotations:
    summary: "Low throughput on {{ $labels.instance }}"

Common Issues & Solutions

Issue Solution
High latency Reduce batch_size, disable compression, increase threads
Low throughput Increase batch_size, enable compression, increase partitions
High memory usage Reduce batch_size, enable compression, reduce cache_size
Storage full Reduce retention_ms, increase segment rotation, add storage
Network saturation Enable compression, reduce message size, increase broker count
