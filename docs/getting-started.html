Getting Started

Prerequisites

Before you begin, ensure you have the following installed:

  • Rust 1.70+ - Download from rustup.rs
  • Go 1.21+ - Download from golang.org
  • Docker & Docker Compose (optional) - For containerized deployment
  • C++17 compiler - For building storage backend
  • CMake 3.15+ - For building C++ components

Installation

1. Clone the Repository

git clone https://github.com/iamrahulgupta/project-streamforge.git
cd project-streamforge

2. Build the Broker

# Build storage backend (C++)
cd storage
mkdir build && cd build
cmake ..
cmake --build . --config Release
cd ../..

# Build broker (Rust)
cd broker
cargo build --release
cd ..

3. Build the SDK and Admin Tools (Go)

cd sdk-go
go mod download
cd ..

cd admin
go mod download
cd ..
Tip: All builds can also use Docker Compose for a containerized setup without local dependencies.

Quick Start with Docker

The easiest way to get started is using Docker Compose, which spins up a complete 3-broker cluster with admin tools and monitoring:

cd docker
docker-compose up -d

This will start:

  • 3 StreamForge brokers (ports 9092, 9093, 9094)
  • Admin REST API (http://localhost:8080)
  • Prometheus (http://localhost:9090)
  • Grafana (http://localhost:3000)

Your First Message

Create a Topic

curl -X POST http://localhost:8080/api/v1/topics \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-topic",
    "partitions": 3,
    "replication_factor": 2
  }'

Produce a Message

curl -X POST http://localhost:8080/api/v1/produce \
  -H "Content-Type: application/json" \
  -d '{
    "topic": "my-topic",
    "key": "user-123",
    "value": "Hello, StreamForge!"
  }'

Consume Messages

curl http://localhost:8080/api/v1/topics/my-topic/partition/0

Using the Go SDK

Producer Example

package main

import (
  "context"
  "log"
  "github.com/iamrahulgupta/streamforge/sdk-go/producer"
)

func main() {
  // Create producer config
  config := producer.DefaultProducerConfig(
    []string{"localhost:9092"},
    "my-topic",
  )

  // Create producer
  p, err := producer.NewProducer(config)
  if err != nil {
    log.Fatalf("Failed to create producer: %v", err)
  }
  defer p.Close()

  // Produce message
  msg := &producer.Message{
    Key:   []byte("user-123"),
    Value: []byte("Hello, StreamForge!"),
  }

  result, err := p.ProduceSync(context.Background(), msg)
  if err != nil {
    log.Fatalf("Failed to produce: %v", err)
  }

  log.Printf("Message produced at offset %d", result.Offset)
}

Consumer Example

package main

import (
  "context"
  "log"
  "github.com/iamrahulgupta/streamforge/sdk-go/consumer"
)

func main() {
  // Create consumer config
  config := consumer.DefaultConsumerConfig(
    []string{"localhost:9092"},
    "my-group",
    []string{"my-topic"},
  )

  // Create consumer
  c, err := consumer.NewConsumer(config)
  if err != nil {
    log.Fatalf("Failed to create consumer: %v", err)
  }
  defer c.Close()

  // Poll messages
  msg, err := c.Poll(context.Background())
  if err != nil {
    log.Fatalf("Failed to poll: %v", err)
  }

  log.Printf("Received: key=%s, value=%s", msg.Key, msg.Value)
  c.CommitMessage(msg)
}

Using the Admin CLI

Build the CLI

cd admin
go build -o streamforge-admin .

Basic Commands

# List all topics
./streamforge-admin topic list

# Create a topic
./streamforge-admin topic create events \
  --partitions 3 \
  --replication-factor 2

# Describe a topic
./streamforge-admin topic describe events

# Produce a message
./streamforge-admin produce \
  --topic events \
  --key "user-123" \
  --value "event data"

# Consume messages
./streamforge-admin consume \
  --topic events \
  --group my-group \
  --max-messages 10

# Start REST API server
./streamforge-admin server --port 8080

Common Issues

Port Already in Use: If ports 9092-9094 are in use, modify docker-compose.yml port mappings or run broker.run on different ports.
Docker Build Fails: Ensure you have sufficient disk space and that Docker daemon is running. Try docker system prune to clean up.
Memory Issues: If compilation fails due to memory, increase Docker's memory limit or build native without Docker.
