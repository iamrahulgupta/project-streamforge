Docker Compose Deployment

Quick Start

cd docker
docker-compose up -d

Verify Deployment

# Check running containers
docker-compose ps

# View logs
docker-compose logs -f

# Test broker health
curl http://localhost:8080/health

Scaling Brokers

To add more brokers, extend docker-compose.yml:

broker-4:
  build:
    context: ..
    dockerfile: docker/broker.Dockerfile
  container_name: streamforge-broker-4
  environment:
    BROKER_ID: 4
    PORT: 9092
    METRICS_PORT: 9101
  ports:
    - "9095:9092"
    - "9104:9101"
  volumes:
    - broker4_data:/data/streamforge
  networks:
    - streamforge-net

Storage Management

# View volumes
docker volume ls | grep streamforge

# Remove all volumes (WARNING: data loss)
docker-compose down -v

# Backup broker data
docker run -v broker1_data:/data -v $(pwd)/backup:/backup \
  alpine tar czf /backup/broker1.tar.gz -C /data .

Kubernetes Deployment

StatefulSet Configuration

apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: streamforge-broker
spec:
  serviceName: streamforge
  replicas: 3
  selector:
    matchLabels:
      app: streamforge
  template:
    metadata:
      labels:
        app: streamforge
    spec:
      containers:
      - name: broker
        image: streamforge:latest
        ports:
        - containerPort: 9092
          name: broker
        - containerPort: 9101
          name: metrics
        env:
        - name: BROKER_ID
          valueFrom:
            fieldRef:
              fieldPath: metadata.name
        - name: LISTEN_ADDR
          value: "0.0.0.0"
        - name: PORT
          value: "9092"
        - name: DATA_DIR
          value: "/data/streamforge"
        volumeMounts:
        - name: data
          mountPath: /data
        resources:
          requests:
            memory: "512Mi"
            cpu: "500m"
          limits:
            memory: "2Gi"
            cpu: "2000m"
        livenessProbe:
          tcpSocket:
            port: 9092
          initialDelaySeconds: 30
          periodSeconds: 10
  volumeClaimTemplates:
  - metadata:
      name: data
    spec:
      accessModes: [ "ReadWriteOnce" ]
      resources:
        requests:
          storage: 100Gi

Service Configuration

apiVersion: v1
kind: Service
metadata:
  name: streamforge
spec:
  clusterIP: None
  selector:
    app: streamforge
  ports:
  - port: 9092
    targetPort: 9092
    name: broker

Helm Chart

Create a Helm chart for easy deployment:

helm repo add streamforge https://charts.streamforge.io
helm install streamforge streamforge/broker \
  --set replicas=3 \
  --set storage.size=100Gi \
  --set broker.config.replication_factor=2

Installation Steps

# 1. Install dependencies
sudo apt-get update
sudo apt-get install -y build-essential cmake git curl

# 2. Clone repository
git clone https://github.com/iamrahulgupta/project-streamforge.git
cd project-streamforge

# 3. Build broker
cd storage && mkdir build && cd build && cmake .. && cmake --build . && cd ../..
cd broker && cargo build --release && cd ..

# 4. Create system user
sudo useradd -r -s /bin/false streamforge

# 5. Create directories
sudo mkdir -p /opt/streamforge/{bin,config,data}
sudo chown -R streamforge:streamforge /opt/streamforge

# 6. Copy binary
sudo cp broker/target/release/broker /opt/streamforge/bin/

# 7. Create systemd service
sudo tee /etc/systemd/system/streamforge.service > /dev/null <

Log Management

# View logs
sudo journalctl -u streamforge -f

# Archive logs
sudo journalctl -u streamforge --since "2 days ago" > broker.log

# Configure log rotation
sudo tee /etc/logrotate.d/streamforge > /dev/null <

Cloud Platform Deployment

AWS

Using EC2 + EBS:

  • Launch m5.2xlarge instances (8 vCPU, 32 GB RAM)
  • Attach gp3 EBS volumes (500 GB, 16000 IOPS)
  • Use placement groups for low latency
  • Place behind NLB (Network Load Balancer) for high performance

Using ECS (Elastic Container Service):

aws ecs create-cluster --cluster-name streamforge

aws ecs register-task-definition \
  --cli-input-json file://task-definition.json

aws ecs create-service \
  --cluster streamforge \
  --service-name broker-service \
  --task-definition streamforge:1 \
  --desired-count 3

GCP

Using GCE (Google Compute Engine):

  • Create n2-standard-8 instances
  • Use pd-ssd persistent disks (500 GB)
  • Create instance group for auto-scaling
  • Use Cloud Load Balancing for traffic distribution

Using GKE (Google Kubernetes Engine):

gcloud container clusters create streamforge \
  --num-nodes 3 \
  --machine-type n2-standard-8 \
  --disk-type pd-ssd \
  --disk-size 500

kubectl apply -f k8s/statefulset.yaml
kubectl apply -f k8s/service.yaml

Azure

Using Azure VMs:

  • Create Standard_D8s_v5 VMs
  • Use Premium SSD (P30, 1TB)
  • Place behind Azure Load Balancer
  • Enable accelerated networking for lower latency

    for: 5m

Backup & Disaster Recovery

Backup Strategy

#!/bin/bash
# Daily backup script

BACKUP_DIR="/backups/streamforge"
mkdir -p $BACKUP_DIR

# Backup each broker
for BROKER_ID in 1 2 3; do
  BROKER_DATA="/data/streamforge-broker-$BROKER_ID"
  BACKUP_FILE="$BACKUP_DIR/broker-$BROKER_ID-$(date +%Y%m%d).tar.gz"
  tar czf $BACKUP_FILE $BROKER_DATA
done

# Upload to cloud storage
aws s3 sync $BACKUP_DIR s3://streamforge-backups/

Recovery Procedure

  1. Stop affected broker: systemctl stop streamforge
  2. Clear data directory: rm -rf /data/streamforge/*
  3. Restore from backup: tar xzf backup-file.tar.gz -C /data
  4. Start broker: systemctl start streamforge
  5. Verify: curl http://localhost:8080/health

Security Considerations

Network Security

  • Use VPN or private networks for inter-broker communication
  • Enable TLS/SSL for all communications
  • Use firewall rules to restrict access
  • Implement network segmentation

Access Control

  • Enable SASL authentication for clients
  • Implement ACLs for topic access
  • Use certificate-based authentication for brokers
  • Rotate credentials regularly

Data Security

  • Encrypt data at rest using dm-crypt or similar
  • Enable TLS for in-transit data
  • Use secure backup storage
  • Implement audit logging
